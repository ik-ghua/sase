## 一、背景

SASE 的"S"(安全)由三项能力承载:**SWG**(出站 Web 安全)、**FWaaS**(防火墙即服务)、**CASB-DLP**(云访问安全 + 数据防泄漏)。L1 3.10 已定它们**挂在同一数据面钩子**(Envoy `ext_authz`/`ext_proc` + eBPF)、**分期上线**(P2/P4/P5)、选择性解密控成本合规。但**各能力内部怎么组织、与策略/风险/解密的接口、起步买什么自研什么**未定义——这是 SASE 区别于纯连接(VPN/SD-WAN)的核心价值所在。

本文给安全栈 L2 总览:三能力如何复用底座、各自内部要点、分期与克制。**它们是分期能力(P2/P4/P5),故先出总览定接口与边界,完整规格留各能力建设时深挖。**

---

## 二、目标(可衡量)

1. **共享底座**:三能力复用同一 PoP + 策略模型 + 数据面钩子,**加能力 = 新策略类型 + 新处理器,不动基础设施**(兑现 L1 "地基统一")。
2. **接口清晰**:与策略编译器(策略类型/`inspect`)、PoP(钩子)、风险引擎(DLP/威胁信号)、解密(国密 TLS)的接口定清。
3. **起步克制**:分类库/威胁情报/DLP 检测**起步采购或规则**,自研 ML 后置(避免过早投入深水区)。
4. **多租户/国密/合规**:三能力按租户隔离、解密走国密(待 PoC-G)、对接数据安全法/PIPL。

**非目标:** 加密算法(PoC-G);Envoy/eBPF 在 PoP 的实现(PoP L2);分类/情报/DLP 自研算法;字节 schema;编码。

## 三、设计

### 3.1 三能力共享底座(加能力不加基础设施)

**背景** L1 3.10 的核心承诺:三能力共享底座。需把"共享什么"定清。

**目标** 明确三能力复用的公共件与扩展点。

**设计 —— 共享件 + 扩展点**
- **共享:** ① PoP 数据面(三件套,PoP L2);② 统一策略模型 + 策略编译器(`主体×资源×动作×条件→效果`,含 `inspect` 效果);③ 数据面钩子——**L3/L4 → eBPF map**、**L7 → Envoy `ext_authz`(策略判定)/`ext_proc`(内容处理)**;④ 遥测/审计、PKI(解密 CA)、多租户隔离。
- **扩展点(加能力的方式):** 新增**策略类型**(编译器)+ 新增 **`ext_proc` 处理器/eBPF 规则类**(数据面)——**不新增基础设施**。
  - SWG/CASB-DLP 主要扩展 `ext_proc` 处理器(内容检查);FWaaS 主要扩展 eBPF(L3/L4)+ Envoy(L7)规则。

依据:统一钩子 + 编译器使"加能力"成为"加策略类型 + 加处理器",复用 PoP/策略/隔离/PKI——这是 L1 "地基统一、能力分期"的工程兑现。
- 备选:每能力独立数据面/网关。落选:重复基础设施、违背地基统一、运维爆炸。

**风险** `ext_proc` 处理器增延迟(内容检查在 L7 热路径)→ 按需启用 + 采样(L1 3.10);处理器质量参差 → 处理器接口规范 + 隔离(一个处理器崩不拖垮 Envoy)。

**结论** 三能力共享 PoP/策略/钩子/PKI/隔离,加能力 = 新策略类型 + 新 `ext_proc`/eBPF 规则类,不动基础设施。

---

### 3.2 SWG(P2):正向代理 / 分类过滤 / 选择性解密 / 恶意拦截

**背景** SWG 是出站 Web 安全网关(L1 3.10),P2(ZTNA 之后第一项安全能力)。

**目标** 定 SWG 内部要点与接口。

**设计**
- **正向代理**:出站流量经 PoP Envoy 正向代理;按 URL/分类/策略放行/拦截/`inspect`。
- **URL/分类过滤**:域名/URL 分类(广告/恶意/赌博/成人等)→ 策略按分类决策。**分类库起步采购**(3.5)。
- **选择性 TLS 解密(成本 + 合规关键)**:**按分类选择性解密**——银行/医疗等豁免、pinned 应用旁路(全量解密既贵又触法律红线,L1 3.10);需端点 Agent 装受信解密 CA。**解密 CA = per-tenant 子 CA + HSM/受保护密钥区**(L1 3.5,命门);**国密客户走国密 TLS 解密**(国密选型 3.4,待 PoC-G)。
- **恶意拦截**:接威胁情报 feed(起步采购,3.5)拦恶意站点;命中 → 拦截 + 告警 + **喂风险引擎**(威胁信号→风险分)。

依据:正向代理 + 选择性解密是 SWG 标准形态;选择性解密平衡成本/合规/可见性(L1 3.10);解密 CA 安全沿用 L1 3.5。
- 备选:全量解密。落选:成本高 + 触法律红线(银行/医疗)+ pinned 应用失败(L1 3.10)。

**风险** 解密隐私/合规 → 选择性 + 豁免 + 审计;解密 CA 在线私钥是高价值目标 → per-tenant 子 CA + HSM 不可导出(L1 3.5/3.12);分类/情报误判 → 阈值可调 + 申诉。

**结论** SWG = 正向代理 + 分类过滤(库采购)+ 选择性解密(豁免敏感/pinned、国密、CA 子 CA+HSM)+ 恶意拦截(情报采购、喂风险);挂 Envoy `ext_authz`。

### 3.3 FWaaS(P4):L3-L7 防火墙 + 每租户分段

**背景** FWaaS = 云交付防火墙 + 租户网络分段(L1 3.10),P4。

**目标** 定 FWaaS 内部与复用关系。

**设计**
- **两级规则(复用策略编译器)**:**L3/L4 → eBPF map**(源/目的/端口/协议,编译器 `L34RuleSet`)、**L7 → Envoy**(应用层规则);复用统一策略模型 + 编译器(`L34RuleSet`/`L7PolicyBundle`),**FWaaS 主要是"新策略类型",基础设施已有**。
- **每租户分段**:租户内网络分段(子网间隔离)经 per-tenant 路由域 + eBPF 规则(L1 3.2 隔离 + PoP L2 3.3);跨段访问按策略。
- **复用度高**:FWaaS 边际成本低(L1 3.10)——eBPF/Envoy 钩子 + 编译器都在,FWaaS 多是策略类型与分段配置。

依据:FWaaS 的 L3/L4 + L7 规则正是我们策略编译器已产的两级产物,复用即可;分段复用租户路由域隔离。
- 备选:独立防火墙数据面。落选:重复 eBPF/Envoy 已有能力。

**风险** 分段规则爆炸(大租户多段)→ 规则编译 + eBPF map 规模(对接编译器/PoP 规模实测);L7 规则与 SWG/策略重叠 → 统一策略模型避免多套。

**结论** FWaaS 复用策略编译器两级产物(eBPF L3/L4 + Envoy L7)+ 租户路由域分段,主要是新策略类型、边际成本低;P4。

### 3.4 CASB / DLP(P5):Inline + API 旁路

**背景** CASB/DLP = 云访问安全代理 + 数据防泄漏(L1 3.10),P5(工程成本最高、依赖 SWG 解密就位)。

**目标** 定 CASB/DLP 两条路径与接口。

**设计 —— 两条路径**
- **Inline DLP(依赖 SWG 解密)**:经 Envoy **`ext_proc` → DLP 引擎**检查出站内容(文件/表单/剪贴板经代理的部分);命中敏感(PII/密级)→ 拦截/告警/`inspect`;**依赖 SWG 选择性解密**(看得到明文才能检,L1 3.10)。
- **API 旁路(out-of-band)**:对接 **企微/钉钉/飞书等 SaaS 的 API**(非 inline)——扫描已存于 SaaS 的数据、分享治理、事后管控;用租户 IdP/授权(身份子 L2),SaaS API 经 **adapter 契约隔离 + 监控**(L1 R6,类身份 IdP adapter)。
- **DLP 检测**:**起步规则为主**(正则/关键字/字典/密级标记),ML/指纹**后置**(3.5);命中**喂风险引擎**(DLP 信号 → 用户/设备风险分,信任风险引擎 3.1)。
- **合规**:对接《数据安全法》《个人信息保护法》(PIPL)——DLP 本身处理敏感数据,需合规留存 + 最小化 + 审计。

依据:Inline(实时拦截,依赖解密)+ API 旁路(覆盖 SaaS 已存数据)是 CASB/DLP 标准两形态;规则起步、ML 后置控成本;命中喂风险引擎闭环(动态访问)。
- 备选:仅 Inline。落选:覆盖不到已在 SaaS 的数据 + 非代理路径;API 旁路补全。
- 备选:起步上 ML/内容指纹。落选:缺标注数据 + 成本高 + 黑盒不可解释(合规),后置。

**风险** Inline DLP 延迟(`ext_proc` 内容检查)→ 按需 + 采样(L1 3.10);DLP 误报/漏报 → 规则 + 阈值可调、人工复核(对接风险引擎对象级标记);SaaS API 变更 → adapter 契约隔离(类身份 IdP adapter);DLP 数据隐私 → 合规留存 + 最小化。

**结论** CASB/DLP = Inline(`ext_proc`→DLP 引擎,依赖 SWG 解密)+ API 旁路(SaaS API,租户授权);DLP 规则起步、ML 后置、命中喂风险引擎;合规对接数据安全法/PIPL;P5。

### 3.5 起步克制:买/规则起步,自研后置

**背景** 分类库、威胁情报、DLP 检测是专业深水区;一上来自研不现实。

**目标** 定哪些起步采购/规则、哪些自研、何时。

**设计**
- **起步采购/外部 feed**:URL/域名分类库、威胁情报 feed(恶意域名/IP)——成熟商业/开源源,**买或订阅**(L1 3.10 "分类库起步买")。
- **起步规则**:DLP 检测以规则为主(正则/关键字/密级/字典);FWaaS/SWG 策略由我们策略模型表达。
- **自研后置**:ML 分类/内容指纹/UEBA 行为分析——数据与成本到位后再做(对接风险引擎 ML 后置)。
- **接口隔离**:分类/情报/DLP 引擎经**适配接口**接入(类身份 IdP adapter),换源/换引擎不动上层——支持"先买后自研"平滑替换。

依据:专业能力起步买/规则可快速上线且可靠;自研留到有数据/规模;接口隔离使替换不返工(与"地基统一"一致)。
- 备选:起步全自研。落选:深水区、慢、质量风险;先买后换更务实。

**风险** 外部源依赖/成本/合规(数据出境?)→ 选国内合规源 + 接口隔离可换;采购源质量 → 多源 + 可换。

**结论** 分类/情报/DLP 起步采购或规则、ML 后置、经适配接口接入(可平滑换自研);先上线、后深化。

### 3.6 多租户、国密与合规

**背景** 三能力都处理租户流量/数据,涉隔离、国密解密、合规。

**目标** 定安全栈的多租户、国密、合规要点。

**设计**
- **多租户**:三能力策略/规则/DLP 标记/审计均按 `tenant_id` 隔离(策略编译器 per-tenant、PoP 租户域、RLS);解密 CA per-tenant 子 CA(L1 3.5)。
- **国密**:SWG 解密链路用国密 TLS(国密选型 3.4,铜锁-Envoy/前置终结,待 PoC-G);国密客户的浏览器/客户端走 TLCP。
- **合规**:SWG 解密 + DLP 处理敏感数据 → 对接《数据安全法》《PIPL》;解密豁免敏感类(银行/医疗);审计 append-only(L1 3.14);数据留存合规(对接 C13 ≥6 个月)。

依据:沿用我们多租户隔离(策略/RLS/路由域/子 CA)、国密选型、合规约束;安全栈不另造隔离。

**风险** 解密 + DLP 的合规红线(隐私)→ 选择性解密 + 豁免 + 最小化 + 审计;国密解密性能 → SM4 受硬件限(PoC-G,同数据面),选择性解密减量。

**结论** 三能力按租户隔离、SWG 解密走国密(待 PoC-G)、合规对接数据安全法/PIPL + 选择性解密豁免敏感 + 审计留存。

## 四、风险

### RS1:`ext_proc` 内容检查增 L7 延迟
缓解:按需启用 + 采样(L1 3.10);处理器隔离(崩不拖垮 Envoy,3.1)。

### RS2:TLS 解密成本/合规/性能
缓解:选择性解密 + 豁免敏感/pinned(3.2);解密 CA per-tenant 子 CA+HSM(L1 3.5);国密解密受 SM4 硬件限(PoC-G)。

### RS3:分类/情报/DLP 起步自研不现实
缓解:起步采购/规则 + 接口隔离可换(3.5),ML 后置。

### RS4:DLP 误报/漏报 + 隐私合规
缓解:规则 + 阈值可调 + 人工复核(风险引擎对象级标记);最小化 + 合规留存(3.6)。

### RS5:SaaS API(CASB 旁路)变更
缓解:adapter 契约隔离 + 监控(类身份 IdP adapter,L1 R6)。

### RS6:三能力策略/规则重叠或冲突
缓解:统一策略模型 + 编译器冲突检测(策略编译器 3.2),不各自一套。

## 五、结论与分期落点

**结论:** 三项安全能力**共享底座**(PoP/统一策略模型/数据面钩子 `ext_authz`+`ext_proc`+eBPF/PKI/隔离),**加能力 = 新策略类型 + 新处理器,不动基础设施**:**SWG(P2)** 正向代理 + 分类过滤(库采购)+ 选择性解密(豁免敏感/pinned、国密、CA 子 CA+HSM)+ 恶意拦截(情报采购、喂风险);**FWaaS(P4)** 复用策略编译器两级产物(eBPF L3/L4 + Envoy L7)+ 租户分段、边际成本低;**CASB/DLP(P5)** Inline(`ext_proc`→DLP 引擎,依赖 SWG 解密)+ API 旁路(SaaS API)、DLP 规则起步 ML 后置、命中喂风险引擎;**起步采购/规则、自研后置、接口隔离可换**;多租户隔离 + 国密解密(待 PoC-G)+ 合规(数据安全法/PIPL + 选择性解密豁免 + 审计)。

**分期落点(各能力建设时深挖子 L2):**
| 能力 | 阶段 | 深挖子 L2(建设时) |
|------|------|---------------------|
| SWG | P2 | 分类/情报源对接、解密链路(国密)、Envoy `ext_authz` 配置 |
| FWaaS | P4 | 分段规则模型、eBPF/Envoy 规则编译细化 |
| CASB/DLP | P5 | DLP 检测引擎、SaaS API 连接器、Inline `ext_proc` 处理器 |

**范围说明:** **安全运营(SecOps:威胁检测响应 / 安全事件编排 / 告警)暂不在本总览**——它是跨能力的运营层,作为**可能的后续 L2**(SS6),与遥测/审计/风险引擎衔接。

**衔接:** 策略编译器(策略类型/inspect/L34RuleSet/L7PolicyBundle)、PoP(钩子落地)、信任风险引擎(DLP/威胁信号→风险)、国密选型(SWG 解密)、身份(CASB API 授权)。

| SS6 | 安全运营(SecOps:威胁检测响应/事件编排/告警)是否单独 L2 | 范围 | 待确认(可作后续 L2) |

> 说明:本总览定三能力共享底座与接口、分期与起步克制(买/规则起步、自研后置);各能力完整规格留分期建设时子 L2。安全运营(SecOps)作为可能的后续 L2 列 SS6。不依赖国密 PoC-G(仅 SWG 解密性能待 M-G3)。
