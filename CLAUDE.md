# CLAUDE.md

本文件为 Claude Code(claude.ai/code)在本仓库工作时提供指引。

## 项目

多租户 **SASE**(安全访问服务边缘)系统,**对外 SaaS 产品**,目标客户为中国大陆 50–5000 人中大型企业(预期租户量级几十到上千)。
- 两条接入轨道:**ZTNA**(替代 VPN,面向人)+ **SD-WAN**(站点组网,面向分支)。
- 三项安全能力:**SWG / FWaaS / CASB-DLP**。
- 核心原则:控制面/数据面分离 + 全国 PoP;**地基统一、能力分期**(两条轨道与安全能力复用同一套底座)。
- 技术栈:**Go 为主**(控制面/数据面/Agent/连接器/CPE);前端 **TS/React**。

## 工作约定(重要)

1. **设计先行、审核后编码。** 架构相关请求优先产出设计说明(图 / 取舍 / 理由),不直接写代码或搭骨架。除非用户明确说"开始编码 / 搭骨架",否则停留在设计与文档层面。
2. **设计文档四要素:** 背景 → 目标 → 风险 → 结论(整篇与各分节都遵循)。
3. **无空话:** 禁用"主流 / 更好 / 健壮 / 显然"等不改变判断、无依据的措辞;每个决策须配 ① 依据 ② 备选及落选原因 ③ 可行方案;能量化就量化;**没有的数据标注"待确认 / 待实测",绝不编造。**
